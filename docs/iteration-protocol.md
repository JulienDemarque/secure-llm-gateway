# Iteration protocol

Implementation must follow small, reviewable increments.

## Rules

1. Keep each iteration scoped to one cohesive change.
2. End every iteration with:
   - change summary
   - rationale
   - suggested next step
3. Require verification evidence for touched behavior (tests or explicit check output).
4. Do not batch unrelated concerns in one iteration.

## Review loop template

- Change summary: what changed and where.
- Rationale: why this is the right next increment.
- Suggested next step: smallest follow-up change to continue safely.
- Risks/open questions: what still needs approval or data.
